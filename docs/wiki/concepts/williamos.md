# WilliamOS

WilliamOS is a private Primary/Owner operating environment.

It organizes goals, loops, Work Orders, evidence, authority, memory, traces, Academy lessons, and Wiki references so the Primary can operate complex systems without losing governance.

## What It Is

- A command environment for governed work.
- A Work Order and evidence system.
- A read-only knowledge and authority surface unless scoped otherwise.
- A way to keep Codex operating bounded loops without making the Owner a courier.

## What It Is Not

- Not SaaS.
- Not an autonomous runtime.
- Not a hidden worker system.
- Not a permission engine that grants itself authority.
- Not a replacement for Primary decisions.
