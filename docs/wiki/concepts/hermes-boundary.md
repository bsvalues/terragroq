# Hermes Boundary

Hermes is the WilliamOS sidecar boundary for future worker capability. It remains disabled unless a separate Owner-authorized Work Order grants a bounded activation path.

Hermes is not active in the current system. It cannot execute commands, call MCP tools, run workers, schedule jobs, mutate memory, touch production, change auth, change DB/schema/env/package/Vercel settings, or bypass the Primary.

Hermes can be documented, reviewed, blocked, proposed, and taught. Future activation would require a Worker Packet, Activation Review Packet, Authority Registry gate, Evidence, Trace requirements, rollback plan, expiration, and explicit Owner authority.
