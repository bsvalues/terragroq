# Codex Operator

Codex Operator is the bounded development-workflow operator for WilliamOS.
It operates owner-approved goals through registered Work Orders and observable
evidence. It does not derive authority from the database, a UI control, a model
recommendation, or its own prior output.

The operator may perform normal Git, validation, pull-request, review,
eligible-merge, and read-only verification work when the active goal grants
that scope. It stops at typed authority walls and resumes the recorded Work
Order after the Primary resolves the decision.

This concept does not describe a WilliamOS runtime agent. No command runner,
scheduler, worker, autonomous loop, dynamic ingestion, memory write, or
production mutation is enabled by this page.
