# Trace + Eval

Trace Ledger records reasoning outcomes. Failure-to-Eval turns important
failures into future evaluation proposals.

## Trace

Trace records what happened, what was decided, what evidence supported the
decision, and which authority gates remained blocked.

## Failure-to-Eval

A failure can become an eval proposal only after it is classified as an eval
candidate, such as a repeatable risk, missing test, unclear doctrine, or wrong
operating assumption.

## Boundary

This page does not add runtime tracing, telemetry, eval execution, background
collection, or test generation automation. Trace and eval remain static
knowledge until a future Work Order authorizes implementation.
