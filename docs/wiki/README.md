# WilliamOS Wiki

The Wiki is the governed reference layer for WilliamOS.

It records concepts, policies, playbooks, glossary terms, and troubleshooting guidance. It is static and read-only unless a future Work Order explicitly scopes changes.

## Purpose

- Define WilliamOS concepts in plain language.
- Record policies and operating boundaries.
- Preserve playbooks that Codex and the Primary can follow.
- Support Academy lessons with stable reference pages.

## Boundaries

- Wiki does not ingest dynamically.
- Wiki does not scan files or call GitHub APIs.
- Wiki does not mutate Work Orders, Evidence, Memory, Authority, or runtime state.
- Wiki does not authorize action.
- Wiki does not replace owner authority.

## Ownership

The Primary owns canon. Wiki pages are useful reference material, but current owner instructions and Work Order packets remain controlling authority for active work.

## Navigation

- `concepts/` defines WilliamOS systems and terms.
- `policies/` records safety and governance policies.
- `playbooks/` records reusable operating procedures.
- `troubleshooting/` records safe diagnosis guidance.
- `glossary.md` defines core vocabulary.

- `concepts/williamos.md` defines the private command environment.

## Knowledge Ownership

The Primary owns canon. Codex may update Wiki knowledge only inside a scoped
Work Order. Wiki pages should preserve boundaries, denied authority, and
evidence expectations beside every concept.
