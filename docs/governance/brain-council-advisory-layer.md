# Brain Council Advisory Layer

Brain Council is a governed advisory layer for WilliamOS. It can inspect doctrine, cite evidence, frame options, explain confidence and risk, prepare decision packets, and recommend Work Orders.

It is not a runtime agent, command runner, autonomous reasoning loop, tool invocation layer, Hermes/MCP activator, memory writer, production writer, or authority bypass.

## Authority

The Primary remains the authority source. Council recommendations require Work Order scope, evidence, validation, stop conditions, and explicit authority before Codex may operate.

## Evidence

Council recommendations must cite the current base, relevant GOAL/WO reports, PR/check state, production verification when making production claims, safety posture, blocked owner decisions, Academy/Wiki doctrine, Hermes boundary doctrine, and WOE evidence rollups.

Missing evidence lowers confidence or blocks the recommendation. Stale evidence must be marked stale.

## Recommendation Flow

1. Frame the problem.
2. Cite context and evidence.
3. List assumptions and options.
4. Assign confidence and risk.
5. State blocked actions and authority required.
6. Recommend a Work Order packet or no action.
7. Name the next safe gate.

The recommendation is not execution. It becomes actionable only after the Owner authorizes a packet and Codex operates inside that packet.

## Blocked And Denied States

Blocked recommendations are missing evidence, owner authority, safe scope, or validation. Denied recommendations cross policy boundaries such as secrets, production writes, auth/DB changes, Hermes/MCP activation, autonomy, memory writes, or TerraFusion/PACS touch.

No background retry, hidden work, notification loop, runtime state engine, or autonomous continuation is created by this doctrine.
