# Safe Hermes Use

Hermes is a governed sidecar concept. It is not active by default, and it is not an autonomous agent.

## What Hermes Is

- A future worker-sidecar boundary.
- A possible receiver of bounded Worker Packets.
- A system that would require Work Orders, Authority, Evidence, Trace, and Owner review before any action.

## What Hermes Is Not

- Not an active worker.
- Not a command runner.
- Not an MCP bridge.
- Not a scheduler or service.
- Not a production writer.
- Not an authority source.
- Not a bypass around Agent Forge, Work Orders, Evidence, Memory, or the Primary.

## Why Disabled By Default Matters

Disabled by default keeps WilliamOS owner-centered. A future worker must not exist just because it is technically possible. It must be needed, scoped, evidenced, bounded, reversible, and authorized.

## Worker Packets

A Worker Packet explains a proposed task, allowed actions, blocked actions, required inputs, expected outputs, authority required, evidence required, safety class, rollback plan, and stop conditions. A Worker Packet is not permission and does not execute.

## Activation Review

Activation Review asks whether a packet is ready for Owner authority. It checks requested capabilities, permissions, allowed surfaces, blocked surfaces, risk, evidence, tests, secrets, production writes, rollback, logging, owner decision, approval state, and expiration.

## Safe Requests

- "Draft a Worker Packet for read-only documentation review."
- "List the evidence that would be required before Hermes could be considered."
- "Explain why Hermes remains blocked."
- "Prepare a revocation checklist for a future bounded Hermes task."

## Blocked Requests

- "Start Hermes."
- "Run this command through Hermes."
- "Let Hermes call MCP tools."
- "Have Hermes deploy production."
- "Let Hermes read secrets."
- "Keep Hermes running in the background."

## Reading Denied / Blocked States

Blocked means protected. It means WilliamOS needs authority, evidence, safer scope, or a different Work Order before the lane can proceed.

## Revocation

Future Hermes authority must be revocable. Revoked Hermes stops, returns evidence, and cannot continue work.
