# Codex Operator Goal and Loop

Goal: `GOAL-WOS-CODEX-OPERATOR-001`

Loop: `LOOP-WOS-CODEX-OPERATOR-001`

The Primary states the desired outcome, grants the risk envelope, and retains
consequential authority. Codex owns routine repository operation inside that
grant. The Owner is not the branch creator, test runner, pull-request monitor,
review messenger, merge operator, or prompt courier.

`/goal` defines intent, success, boundaries, risk ceiling, and standing
authority. `/loop` repeatedly selects dependency-ready Work Orders, refreshes
current truth, validates, remediates, merges eligible work, records evidence,
and continues. A Work Order owns one bounded mutation or proof task. `/stop`
records a typed authority wall and the exact resume action.

Normal failures stay with Codex when the repair remains in scope. Auth, secrets,
schemas, production writes, external-system changes, runtime activation,
destructive operations, and material scope expansion return to the Primary as
one decision-ready packet.

This curriculum is static. It does not create a command runner, scheduler,
worker, background loop, memory writer, dynamic ingestion path, production
writer, or autonomous product behavior.
