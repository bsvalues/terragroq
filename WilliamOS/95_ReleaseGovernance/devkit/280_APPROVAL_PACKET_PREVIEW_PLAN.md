# Phase 5W Approval Packet Preview Plan

## Purpose

Add a preview-only Approval Packet composer that packages owner decision drafts
into copyable approval packet text and required evidence.

## Scope

Allowed implementation scope:

- `control-center/backend/approval_packet_preview.py`
- `GET /api/approval-packet-preview`
- Control Center GUI preview/copy panel
- backend tests
- devkit documentation and validation report
- matching tracked frontend `dist/` output if regenerated by a passing build

## Safety Boundary

The approval packet is preview/copy only.

It must not:

- approve packets
- grant authority
- write decision records
- record approval
- change state
- execute commands
- execute validators
- stage or commit
- push, open PRs, merge, release, or tag
- schedule work
- activate MCP
- enable autonomy
- write production data

## API

`GET /api/approval-packet-preview`

Returns:

- approval packet previews
- approval text
- required evidence
- missing authorities
- denied actions
- safety flags proving preview-only behavior

No POST, PUT, PATCH, DELETE, approve, grant, persist, run, execute, schedule, or
activate endpoint is part of this phase.

## GUI

The Control Center shows an Approval Packet panel with:

- packet count
- owner-required count
- approval/grant safety flags
- packet preview cards
- clipboard-only copy buttons
- denied actions

The panel writes no packet files.

## Validation

Required checks:

- focused backend tests for approval packet preview
- full backend test suite
- frontend build
- safety scan for approval/grant/write/execution/autonomy/MCP/production-write paths
- scope review before local commit
