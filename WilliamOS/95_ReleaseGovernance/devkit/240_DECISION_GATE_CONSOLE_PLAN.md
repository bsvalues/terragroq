# Phase 5S Decision Gate Console Plan

## Purpose

Add a preview-only Decision Gate Console that summarizes the owner decision
needed next from the existing governed operator surfaces.

The console composes:

- repo state dashboard
- commit readiness review
- operator review inbox
- validation runbook registry
- active work order registry

## Scope

Allowed implementation scope:

- `control-center/backend/decision_gate_console.py`
- `GET /api/decision-gate-console`
- Control Center GUI preview panel
- backend tests
- devkit documentation and validation report
- matching tracked frontend `dist/` output if regenerated by a passing build

## Safety Boundary

The Decision Gate Console is decision support only.

It must not:

- approve gates
- change state
- persist decisions
- execute work
- execute validators
- stage or commit
- push, open PRs, merge, release, or tag
- schedule work
- activate MCP
- enable autonomy
- write production data

## API

`GET /api/decision-gate-console`

Returns:

- pending owner decisions
- blocked gates
- allowed next actions
- denied actions
- recommended work order lane
- next valid gate
- safety flags proving preview-only behavior

No POST, PUT, PATCH, DELETE, run, execute, approve, persist, or activate
endpoint is part of this phase.

## GUI

The Control Center shows a Decision Gates panel with:

- gate status
- current HEAD
- pending owner decisions
- blocked gates
- recommended WO lane
- allowed next actions
- denied actions
- preview-only safety flags

The panel has a refresh button only.

## Validation

Required checks:

- focused backend tests for decision gate console
- full backend test suite
- frontend build
- safety scan for write/execute/persistence/autonomy/MCP/production-write paths
- scope review before local commit
