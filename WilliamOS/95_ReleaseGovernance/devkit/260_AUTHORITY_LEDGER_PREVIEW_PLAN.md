# Phase 5U Authority Ledger Preview Plan

## Purpose

Add a preview-only Authority Ledger that shows what authority exists, what
authority is missing, who must approve next, and which actions remain denied for
each proposed lane/action route.

## Scope

Allowed implementation scope:

- `control-center/backend/authority_ledger_preview.py`
- `GET /api/authority-ledger`
- Control Center GUI preview panel
- backend tests
- devkit documentation and validation report
- matching tracked frontend `dist/` output if regenerated by a passing build

## Safety Boundary

The ledger is authority visibility only.

It must not:

- grant authority
- record approval
- write state
- execute commands
- execute validators
- stage or commit
- push, open PRs, merge, release, or tag
- schedule work
- activate MCP
- enable autonomy
- write production data

## API

`GET /api/authority-ledger`

Returns:

- authority entries derived from action routes
- current authority state
- missing authority
- required approver
- denied actions
- recommended work order lane
- next valid gate
- safety flags proving preview-only behavior

No POST, PUT, PATCH, DELETE, approve, grant, persist, run, execute, schedule, or
activate endpoint is part of this phase.

## GUI

The Control Center shows an Authority Ledger panel with:

- entry count
- missing authority count
- grant/record safety flags
- current authority per route
- missing authority per route
- required approver
- denied actions

The panel has a refresh button only.

## Validation

Required checks:

- focused backend tests for authority ledger
- full backend test suite
- frontend build
- safety scan for write/execute/grant/persistence/autonomy/MCP/production-write paths
- scope review before local commit
