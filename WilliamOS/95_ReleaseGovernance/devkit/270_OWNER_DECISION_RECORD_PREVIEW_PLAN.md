# Phase 5V Owner Decision Record Preview Plan

## Purpose

Add a preview-only Owner Decision Record composer that drafts what an owner
decision record would contain if the owner later authorizes recording it.

## Scope

Allowed implementation scope:

- `control-center/backend/owner_decision_record_preview.py`
- `GET /api/owner-decision-record-preview`
- Control Center GUI preview panel
- backend tests
- devkit documentation and validation report
- matching tracked frontend `dist/` output if regenerated by a passing build

## Safety Boundary

The preview is draft-only.

It must not:

- write decision records
- grant authority
- record approval
- change state
- execute commands
- execute validators
- stage or commit
- push, open PRs, merge, release, or tag
- schedule work
- activate MCP
- enable autonomy
- write production data

## API

`GET /api/owner-decision-record-preview`

Returns:

- decision record drafts
- owner-required count
- missing authorities
- recommended record text
- evidence references
- denied actions
- safety flags proving preview-only behavior

No POST, PUT, PATCH, DELETE, approve, grant, persist, run, execute, schedule, or
activate endpoint is part of this phase.

## GUI

The Control Center shows an Owner Decision Preview panel with:

- draft count
- owner-required count
- write/grant safety flags
- draft decision cards
- approver and authority requirements
- denied actions

The panel has a refresh button only.

## Validation

Required checks:

- focused backend tests for owner decision record preview
- full backend test suite
- frontend build
- safety scan for write/grant/approval/execution/autonomy/MCP/production-write paths
- scope review before local commit
