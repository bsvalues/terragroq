# Phase 5T Operator Action Router Preview Plan

## Purpose

Add a preview-only Operator Action Router that maps pending decisions and work
order gates to safe next-action categories.

Supported categories:

- hold
- review
- validate
- split
- commit-authorize
- push-authorize
- PR-authorize
- stop

## Scope

Allowed implementation scope:

- `control-center/backend/operator_action_router_preview.py`
- `GET /api/operator-action-router`
- Control Center GUI preview panel
- backend tests
- devkit documentation and validation report
- matching tracked frontend `dist/` output if regenerated by a passing build

## Safety Boundary

The router is classification only.

It must not:

- perform the routed action
- approve gates
- write state
- execute commands
- execute validators
- stage or commit
- push, open PRs, merge, release, or tag
- schedule work
- activate MCP
- enable autonomy
- write production data

## API

`GET /api/operator-action-router`

Returns:

- action type catalog
- route previews for current pending decisions
- authority required for each route
- recommended action type
- recommended work order lane
- next valid gate
- safety flags proving preview-only behavior

No POST, PUT, PATCH, DELETE, run, execute, approve, persist, schedule, or
activate endpoint is part of this phase.

## GUI

The Control Center shows an Action Router panel with:

- route count
- recommended action type
- recommended WO lane
- next valid gate
- decision-to-action route cards
- authority required per route
- denied actions
- preview-only safety flags

The panel has a refresh button only.

## Validation

Required checks:

- focused backend tests for operator action router
- full backend test suite
- frontend build
- safety scan for write/execute/persistence/autonomy/MCP/production-write paths
- scope review before local commit
